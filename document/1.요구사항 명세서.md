# 요구사항 명세서

## 개요
- 반려동물 보호자가 수의사에게 비대면으로 상담을 받을 수 있도록 지원하는 수의사 온라인 상담 플랫폼.

## 기능 요구사항

### 보호자(User)

| 요구사항 ID | 요구사항 내용                                     |
|---------|---------------------------------------------|
| F-U-01  | 보호자는 회원가입 및 로그인을 할 수 있어야 한다.                |
| F-U-02  | 보호자는 반려동물 프로필을 등록·수정·삭제할 수 있어야 한다.          |
| F-U-03  | 보호자는 상담을 진행할 수 있는 수의사 리스트를 검색할 수 있어야 한다.    |
| F-U-04  | 보호자는 상담을 진행할 수 있는 수의사의 기본 정보를 조회할 수 있어야 한다. |
| F-U-05  | 보호자는 증상 및 카테고리를 입력하여 상담을 요청할 수 있어야 한다.      |
| F-U-06  | 보호자는 수의사와 실시간 채팅 상담을 진행할 수 있어야 한다.          |
| F-U-07  | 보호자는 상담 종료를 요청할 수 있어야 한다.                   |
| F-U-08  | 보호자는 상담 이력을 조회할 수 있어야 한다.                   |

### 수의사(Vet)

| 요구사항 ID | 요구사항 내용 |
|--------|--------------|
| F-V-01 | 수의사는 자격 정보를 등록하고 인증을 요청할 수 있어야 한다. |
| F-V-02 | 수의사는 매칭된 상담 요청을 수신할 수 있어야 한다. |
| F-V-03 | 수의사는 보호자와 실시간 채팅 상담을 진행할 수 있어야 한다. |
| F-V-04 | 수의사는 상담을 종료 처리할 수 있어야 한다. |
| F-V-05 | 수의사는 상담 종료 후 상담 리포트를 작성할 수 있어야 한다. |

### 관리자(Admin)

| 요구사항 ID | 요구사항 내용 |
|----------|--------------|
| F-A-01 | 관리자는 수의사 자격을 검증하고 승인할 수 있어야 한다. |
| F-A-02 | 관리자는 상담 카테고리 및 태그를 관리할 수 있어야 한다. |
